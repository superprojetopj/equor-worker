Equor Worker

AI-Powered Legal Document Processing Microservice

Node.js 22 TypeScript Fastify 5 Claude AI Google Cloud Storage
Request Flow
1

Receive Task

PHP backend sends a POST request with taskId and processId. The worker authenticates via shared secret header and returns 202 Accepted immediately.

POST /task → 202 Accepted
2

Fetch & Validate

Fetches process data from the backend API. The response is validated with Zod — no unsafe type assertions. Downloads context files (PDFs, contracts) from Google Cloud Storage into memory.

backend.service.ts → storage.service.ts
3

Extract Placeholders

Scans the HTML template for {{PROMPT: instruction}} markers. Each placeholder contains an AI instruction that tells Claude what content to generate for that spot.

html-parser.ts → extractPromptPlaceholders()
4

AI Generation

For each placeholder, sends the instruction + context files + current HTML to Claude. The AI generates formal Brazilian legal text. Placeholders are filled sequentially so each one sees the updated document.

claude.service.ts → callClaude()
5

Report Result

Sends the completed HTML back to the backend with status completed. If anything fails, reports failed with the error message. Multiple documents are processed in parallel — one failure doesn't cancel the others.

reportDocumentResult() → POST /result
External Services & Timeouts
⚙

PHP Backend (Yii2)

Fetches process data and reports results. Authenticated via shared secret header. Response validated with Zod schema.
⏲ 30s timeout
✨

Claude AI (Anthropic)

Generates formal legal text in Brazilian Portuguese. Receives PDFs, metadata, and HTML context. Returns only the content, no explanations.
⏲ 10min timeout
☁

Google Cloud Storage

Downloads context files (contracts, powers of attorney) directly into memory as base64. No temporary files on disk.
⏲ 60s timeout
Reliability & Safety
■

Graceful Shutdown

When Cloud Run sends SIGTERM (deploy, scale-down), the worker stops accepting new tasks (returns 503), waits up to 25 seconds for in-flight tasks to finish, then exits cleanly. No orphaned documents stuck in "processing" forever.

✓

Zod Validation

All data from external sources is validated at runtime with Zod schemas. If the backend returns an unexpected shape, the error is caught immediately with a clear message — not a cryptic crash later.

⚠

Fault-Tolerant Errors

If reporting a failure to the backend also fails (e.g. network down), the error is logged instead of crashing. Each document is isolated via Promise.allSettled — one failure never cancels the others.

🔒

Container Hardening

Runs as non-root user inside Docker. NODE_ENV=production set at build time. Built-in HEALTHCHECK pings /health every 30 seconds. Alpine-based image for minimal attack surface.
